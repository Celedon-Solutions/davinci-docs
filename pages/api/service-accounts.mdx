## Overview

Service accounts are non-human identities used by automation scripts, CI/CD pipelines, and AI agents to interact with the Davinci Programmatic API. Each service account is tied to a human user account for billing purposes.

Service accounts are managed with a **JWT access token** or an existing **API key** from a human user account.

## Authentication

All service account management endpoints (except `GET /meta/scopes`) require authentication via either:

- **JWT Bearer token** from `POST /api/v2/auth/signin`
- **API key** (for read operations; create/update/delete/key-management are restricted to human users)

```
Authorization: Bearer <your_access_token_or_api_key>
```

## Create Service Account

Creates a new service account under your user.

```
POST /api/v1/service-accounts
```

### Request Body

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | Yes | Display name for the service account. Max 100 characters. |

### Example

<CodeGroup>

```bash curl
curl -X POST "https://your-davinci-instance.com/api/v1/service-accounts" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "CI/CD Pipeline Bot"
  }'
```

```typescript JavaScript SDK
import { DavinciClient } from '@celedon/davinci-sdk';

// Note: Service account management requires a JWT token,
// not an API key. Use a direct HTTP request:
const response = await fetch(
  'https://your-davinci-instance.com/api/v1/service-accounts',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({ name: 'CI/CD Pipeline Bot' }),
  }
);

const { data } = await response.json();
console.log(`Service account created: ${data.licenseKey}`);
```

```python Python SDK
import httpx

async with httpx.AsyncClient() as client:
    response = await client.post(
        "https://your-davinci-instance.com/api/v1/service-accounts",
        headers={
            "Authorization": "Bearer YOUR_ACCESS_TOKEN",
            "Content-Type": "application/json",
        },
        json={"name": "CI/CD Pipeline Bot"},
    )

    data = response.json()["data"]
    print(f"Service account created: {data['licenseKey']}")
```

</CodeGroup>

### Response `201 Created`

```json
{
  "success": true,
  "data": {
    "licenseKey": "sa_abc123def456",
    "accountType": "service",
    "serviceAccountName": "CI/CD Pipeline Bot",
    "parentLicenseKey": "usr_your_license_key",
    "createdBy": "usr_your_license_key",
    "createdAt": "2026-02-10T12:00:00.000Z",
    "apiKeys": []
  }
}
```

### Errors

| Code | Status | Description |
|------|--------|-------------|
| `INVALID_NAME` | 400 | Name is missing, empty, or not a string |
| `NAME_TOO_LONG` | 400 | Name exceeds 100 characters |
| `FORBIDDEN` | 403 | Service accounts cannot create other service accounts |
| `CREATE_FAILED` | 400 | Failed to create service account |

## Create API Key

Creates a new API key for a service account.

<Warning>**The full API key is only returned once.** Store it immediately in a secure location.</Warning>

```
POST /api/v1/service-accounts/:id/keys
```

### Path Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `id` | string | Yes | The service account license key |

### Request Body

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `name` | string | No | Display name for the key (default: "Default Key") |
| `scopes` | string[] | No | Permission scopes (default: all scopes) |
| `rateLimit` | number | No | Requests per minute, 1-10,000 (default: 60) |
| `ipAllowlist` | string[] | No | Allowed IP addresses (default: any) |
| `expiresAt` | string | No | ISO 8601 expiration date (default: never) |

### Available Scopes

| Scope | Description |
|-------|-------------|
| `projects:read` | Read project data (tree, objects) |
| `projects:write` | Create/delete projects |
| `sessions:manage` | Create and manage sessions |
| `agent:message` | Send messages to Davinci agent |
| `webhooks:receive` | Required for webhook delivery |

### Example

<CodeGroup>

```bash curl
curl -X POST "https://your-davinci-instance.com/api/v1/service-accounts/sa_abc123def456/keys" \
  -H "Authorization: Bearer YOUR_ACCESS_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Production Key",
    "scopes": ["projects:read", "projects:write", "sessions:manage", "agent:message"],
    "rateLimit": 120,
    "expiresAt": "2027-02-10T00:00:00.000Z"
  }'
```

```typescript JavaScript SDK
const response = await fetch(
  'https://your-davinci-instance.com/api/v1/service-accounts/sa_abc123def456/keys',
  {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      name: 'Production Key',
      scopes: ['projects:read', 'projects:write', 'sessions:manage', 'agent:message'],
      rateLimit: 120,
      expiresAt: '2027-02-10T00:00:00.000Z',
    }),
  }
);

const { data } = await response.json();
// IMPORTANT: Store this key securely - it won't be shown again!
console.log(`API Key: ${data.key}`);
```

```python Python SDK
import httpx

async with httpx.AsyncClient() as client:
    response = await client.post(
        "https://your-davinci-instance.com/api/v1/service-accounts/sa_abc123def456/keys",
        headers={
            "Authorization": "Bearer YOUR_ACCESS_TOKEN",
            "Content-Type": "application/json",
        },
        json={
            "name": "Production Key",
            "scopes": ["projects:read", "projects:write", "sessions:manage", "agent:message"],
            "rateLimit": 120,
            "expiresAt": "2027-02-10T00:00:00.000Z",
        },
    )

    data = response.json()["data"]
    # IMPORTANT: Store this key securely - it won't be shown again!
    print(f"API Key: {data['key']}")
```

</CodeGroup>

### Response `201 Created`

```json
{
  "success": true,
  "data": {
    "keyId": "key_xyz789",
    "keyPrefix": "dav_sk_live_abc1",
    "name": "Production Key",
    "key": "dav_sk_live_abc123def456ghi789jkl012mno345",
    "createdAt": "2026-02-10T12:01:00.000Z",
    "lastUsedAt": null,
    "expiresAt": "2027-02-10T00:00:00.000Z",
    "scopes": ["projects:read", "projects:write", "sessions:manage", "agent:message"],
    "rateLimit": 120,
    "ipAllowlist": [],
    "isActive": true
  },
  "warning": "Store this API key securely. It will not be shown again."
}
```

### Errors

| Code | Status | Description |
|------|--------|-------------|
| `FORBIDDEN` | 403 | Only human users who own the account can create keys |
| `NOT_FOUND` | 404 | Service account does not exist |
| `INVALID_SCOPES` | 400 | Invalid scope values provided |
| `INVALID_RATE_LIMIT` | 400 | Rate limit must be between 1 and 10,000 |
| `INVALID_IP_ALLOWLIST` | 400 | IP allowlist must be an array of strings |
| `INVALID_EXPIRATION` | 400 | Expiration must be a valid future ISO 8601 date |
| `CREATE_KEY_FAILED` | 400 | Failed to create API key |

```

```

### Path Parameters

| Parameter | Type | Required | Description |

<CodeGroup>
