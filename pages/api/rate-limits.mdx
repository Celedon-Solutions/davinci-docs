---
title: Rate Limits
description: Programmatic API rate limit behavior
---

API keys have per-key request limits. When a key is over its limit, the API returns `429 RATE_LIMITED` and includes `Retry-After` with the number of seconds to wait before retrying.

```http
HTTP/1.1 429 Too Many Requests
Retry-After: 5
```

SDKs surface this as `RateLimitError` with a retry-after value.
